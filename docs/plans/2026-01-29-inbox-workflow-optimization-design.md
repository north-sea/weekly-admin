# 收件箱工作流优化设计

## 背景

当前周刊内容处理流程：RSS/Karakeep → 收件箱 → 手动晋升 → 内容池 → 周刊编辑 → 发布

**现状**：
- 内容来源：Karakeep 和 RSS 各占约 50%
- 收件箱阶段是主要人工瓶颈（质量判断、去重、分类、标签、编辑都需要人工）
- 周刊编辑阶段相对轻松，主要是排序和分组
- 截图质量不稳定，尺寸难以保证

**挑战**：
- RSS 接入后，候选量增大，入选率下降
- RSS 内容需要手动截图

## 目标

在保证周刊质量的前提下，降低人工处理时间

## 设计方案

### 一、收件箱智能筛选

#### 1.1 AI 预处理（同步时自动执行）

每条新内容入库时自动计算：

```
├── 质量评分 (0-100)
│   ├── 来源可信度（RSS 源历史入选率）
│   ├── 内容完整度（标题、摘要、正文长度）
│   └── 时效性（发布时间）
├── 重复检测
│   ├── URL 去重（已有）
│   └── 标题/内容相似度（标记疑似重复）
├── 分类建议（基于关键词 + 历史数据）
└── 标签建议（Karakeep AI 标签 + 补充推断）
```

#### 1.2 批量审核界面

```
┌─────────────────────────────────────────────────────────────┐
│ 收件箱 - 待审核 (32)        [全选] [批量晋升] [批量拒绝]    │
├─────────────────────────────────────────────────────────────┤
│ ☑ [85分] 《React 19 新特性》     分类:前端  标签:React      │
│   └─ AI建议: 高质量，建议晋升                               │
│ ☑ [82分] 《Rust 异步编程》       分类:后端  标签:Rust       │
│ ☐ [45分] 《10个CSS技巧》        分类:前端  ⚠️疑似重复       │
│   └─ 与《CSS实用技巧》相似度 87%                            │
│ ☐ [30分] 《广告软文标题》        分类:未知  ⚠️低质量        │
└─────────────────────────────────────────────────────────────┘
```

**关键交互**：
- 高分内容默认勾选，低分/疑似重复默认不选
- 一键批量晋升勾选项
- 点击可展开查看详情/修改分类标签

**预期效果**：收件箱处理时间减少 50-70%

### 二、截图处理优化

采用「统一裁剪 + 占位图兜底」组合方案。

#### 2.1 截图处理流程

```
内容进入系统时：
├── 有图片（Karakeep 截图 / RSS 配图）
│   ├── 尺寸符合 → 直接使用
│   └── 尺寸不符 → 标记"待裁剪"
└── 无图片
    └── 标记"需截图"或"使用占位图"
```

#### 2.2 批量审核界面增加截图状态

```
┌──────────────────────────────────────────────────────────────┐
│ ☑ [85分] 《React 19》   [✓图片OK]    分类:前端               │
│ ☑ [82分] 《Rust 异步》  [⚠️待裁剪]   分类:后端   [裁剪]      │
│ ☑ [78分] 《Go 并发》    [📷无图]     分类:后端   [上传][占位] │
└──────────────────────────────────────────────────────────────┘
```

#### 2.3 裁剪交互

点击「裁剪」按钮弹出裁剪对话框：

```
┌─────────────────────────────────────┐
│  拖动选择裁剪区域 (16:9)            │
│  ┌─────────────────────────┐        │
│  │ ┌───────────────┐       │        │
│  │ │   选中区域    │       │        │
│  │ └───────────────┘       │        │
│  └─────────────────────────┘        │
│                                     │
│  [取消]              [确认裁剪]     │
└─────────────────────────────────────┘
```

- 固定目标比例（如 16:9）
- 用户拖动选择裁剪区域
- 自动压缩到目标分辨率（如 800x450）

#### 2.4 占位图设计

按分类预设一组简洁的占位图：

| 分类 | 配色 | 图标 |
|------|------|------|
| 前端 | 蓝色调 | 代码图标 |
| 后端 | 绿色调 | 服务器图标 |
| 工具 | 橙色调 | 工具图标 |
| 其他 | 灰色调 | 通用图标 |

**预期效果**：不再纠结截图问题，保证视觉一致性

### 三、RSS 源质量管理

#### 3.1 源质量追踪

每个 RSS 源自动统计：

```
├── 总抓取数
├── 晋升数（入选内容池）
├── 入选率 = 晋升数 / 总抓取数
└── 最终入刊率 = 入刊数 / 晋升数
```

#### 3.2 源管理界面

```
┌─────────────────────────────────────────────────────────────┐
│ 数据源管理                                                   │
├─────────────────────────────────────────────────────────────┤
│ 源名称          类型      入选率    入刊率    状态    操作   │
├─────────────────────────────────────────────────────────────┤
│ 阮一峰周刊      RSS       85%       70%      启用    [配置] │
│ Hacker News     RSS       45%       30%      启用    [配置] │
│ 某低质量源      RSS       5%        2%       ⚠️      [禁用] │
│ Karakeep        书签      95%       80%      启用    [配置] │
└─────────────────────────────────────────────────────────────┘
```

#### 3.3 源级别配置

```
┌─────────────────────────────────────┐
│ 配置: 阮一峰周刊                     │
├─────────────────────────────────────┤
│ 默认分类: [前端 ▼]                  │
│ 默认标签: [周刊] [精选]             │
│ AI 评分加权: [+20] (可信源加分)     │
│ 聚合源模式: [✓] 提取文章中的链接    │
│ 同步频率: [每天 ▼]                  │
└─────────────────────────────────────┘
```

**关键点**：
- 高质量源的内容自动获得评分加成，在批量审核时排在前面
- 低质量源可以降低权重或直接禁用
- 聚合源（周刊类）自动提取链接，避免只抓到一条"周刊汇总"

**预期效果**：减少低质量内容噪音

## 简化后的流程

```
RSS/Karakeep
    ↓ 自动同步
收件箱（AI 预处理：评分、去重、分类、标签）
    ↓ 批量审核（勾选 + 一键晋升）
内容池（截图裁剪/占位图处理）
    ↓ 周刊编辑器选材
周刊发布
```

## 变更点

### 去掉的功能
- 自动晋升逻辑（改为人工批量确认，更可控）

### 数据库变更

#### inbox_items 表新增字段
- `ai_score_details` JSON - AI 评分明细（来源分、完整度分、时效分）
- `similar_item_id` INT - 疑似重复的目标项 ID
- `similarity_score` FLOAT - 相似度分数
- `image_status` ENUM('ok', 'needs_crop', 'missing') - 截图状态

#### data_sources 表新增字段
- `score_weight` INT DEFAULT 0 - AI 评分加权值
- `total_synced` INT DEFAULT 0 - 总抓取数
- `total_promoted` INT DEFAULT 0 - 晋升数
- `total_published` INT DEFAULT 0 - 入刊数

## 实施优先级

1. **P0 - 批量审核界面**：最直接解决当前痛点
2. **P1 - AI 预评分**：提升批量审核效率
3. **P1 - 截图裁剪功能**：解决截图尺寸问题
4. **P2 - 源质量追踪**：RSS 接入后再完善
5. **P2 - 占位图系统**：作为截图的兜底方案

## 预期收益

| 模块 | 当前问题 | 优化方案 | 预期效果 |
|------|----------|----------|----------|
| 收件箱筛选 | 逐条审核，效率低 | AI 预评分 + 批量审核界面 | 处理时间减少 50-70% |
| 截图处理 | 尺寸不一，质量不稳定 | 统一裁剪 + 分类占位图兜底 | 不再纠结截图问题 |
| RSS 源管理 | 无质量追踪 | 源级别统计 + 权重配置 | 减少低质量内容噪音 |
