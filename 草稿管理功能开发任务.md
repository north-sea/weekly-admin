# 草稿管理功能开发任务清单

## 📋 项目概述

在内容管理下新增"草稿管理"功能，用于从 Karakeep 同步书签数据，作为周刊内容的候选池。

## 🎯 核心需求

1. 从 Karakeep API 同步书签数据到数据库
2. 支持增量同步（已存在数据以 Karakeep 为准更新）
3. 支持草稿状态管理（待处理/已采用/已拒绝）
4. 支持将草稿转换为正式内容
5. 支持 AI 建议分类
6. 支持去重检测（以 source_url 为准）
7. 丰富的搜索和筛选条件

## 📊 数据库设计

### drafts 表结构

```sql
CREATE TABLE drafts (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  
  -- Karakeep 原始数据
  karakeep_id VARCHAR(100) UNIQUE NOT NULL COMMENT 'Karakeep书签ID',
  title VARCHAR(500) NOT NULL COMMENT '标题',
  url VARCHAR(1000) NOT NULL COMMENT '原始链接',
  description TEXT COMMENT '摘要/描述',
  note TEXT COMMENT '笔记',
  favicon_url VARCHAR(500) COMMENT '网站图标',
  image_url VARCHAR(500) COMMENT '缩略图',
  
  -- Karakeep 元数据
  karakeep_created_at TIMESTAMP COMMENT 'Karakeep创建时间',
  karakeep_updated_at TIMESTAMP COMMENT 'Karakeep更新时间',
  
  -- 扩展字段 - 用于草稿管理
  status ENUM('pending', 'adopted', 'rejected') DEFAULT 'pending' COMMENT '采用状态',
  priority INT DEFAULT 0 COMMENT '优先级',
  category_suggestion VARCHAR(100) COMMENT '建议分类',
  tags_suggestion JSON COMMENT '建议标签（数组）',
  
  -- 去重和关联
  duplicate_of_draft_id BIGINT COMMENT '重复的草稿ID',
  content_id BIGINT COMMENT '关联的内容ID',
  
  -- 同步信息
  synced_at TIMESTAMP COMMENT '最后同步时间',
  
  -- 系统字段
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  
  -- 索引
  INDEX idx_status (status),
  INDEX idx_karakeep_id (karakeep_id),
  INDEX idx_url (url(255)),
  INDEX idx_synced_at (synced_at),
  INDEX idx_priority (priority),
  FOREIGN KEY (content_id) REFERENCES contents(id) ON DELETE SET NULL,
  FOREIGN KEY (duplicate_of_draft_id) REFERENCES drafts(id) ON DELETE SET NULL
) COMMENT '草稿（Karakeep书签）';
```

## 🔄 数据同步逻辑

### 同步流程
1. 调用 Karakeep API 获取所有书签
2. 遍历书签，对每条记录：
   - 使用 `karakeep_id` 查找是否已存在
   - **如果存在**：以 Karakeep 数据为准更新（保留 status、priority 等扩展字段）
   - **如果不存在**：插入新记录
3. 执行去重检测（以 source_url 为准）
4. 对新记录执行 AI 分类建议
5. 返回同步统计

### 去重规则
- 主要依据：`url`（完整匹配）
- 如果发现重复，标记 `duplicate_of_draft_id` 指向最早的记录
- 在列表中可以选择显示/隐藏重复项

### 配置说明：

需要在 `.env` 文件中配置以下环境变量：
```env
KARAKEEP_HOST="https://your-karakeep-instance.com"
KARAKEEP_KEY="your_karakeep_api_key"
KARAKEEP_DRAFT_LIST_ID="your_draft_list_id"  # 新增
```

### 使用步骤：

1. 在 Karakeep 中创建一个名为 "Draft" 的列表
2. 获取该列表的 ID（可以从 Karakeep 的 URL 或 API 中获取）
3. 将列表 ID 配置到 `KARAKEEP_DRAFT_LIST_ID` 环境变量
4. 在 Karakeep 中将需要同步的书签添加到 Draft 列表
5. 在 Weekly 草稿管理页面点击"同步"按钮
