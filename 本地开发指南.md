# 🛠️ 本地开发环境配置指南

本指南详细说明如何在本地搭建Weekly内容管理系统的开发环境。

## 📋 前置要求

### 必需软件
- **Node.js** 18+ 
- **pnpm** (推荐) 或 npm/yarn
- **Git**

### 外部服务
确保以下服务正在运行：
- **MySQL数据库**：NAS服务器 (100.113.231.101:3306)
- **Meilisearch容器**：karakeep-meilisearch (localhost:7700)

## 🚀 快速开始

### 1. 克隆项目

```bash
git clone <repository-url>
cd weekly-admin
```

### 2. 安装依赖

```bash
# 推荐使用pnpm
pnpm install

# 或使用npm
npm install
```

### 3. 配置环境变量

```bash
# 复制开发环境配置模板
cp env.development.example .env

# 编辑配置文件
vim .env
```

**最小配置示例：**
```env
# 数据库连接
DATABASE_URL="mysql://weekly_user:weekly_20250629_@NAS@100.113.231.101:3306/weekly_blog"

# 搜索引擎
MEILISEARCH_HOST="http://localhost:7700"
MEILISEARCH_MASTER_KEY="your_master_key"

# JWT认证
JWT_SECRET="dev-jwt-secret-key"
JWT_EXPIRES_IN="24h"

# 环境设置
NODE_ENV="development"
PORT="3000"
```

### 4. 初始化数据库

```bash
# 生成Prisma客户端
pnpm db:generate

# 如果需要同步数据库结构
pnpm db:pull
```

### 5. 启动开发服务器

```bash
# 标准启动
pnpm dev

# 使用Turbopack（更快的热重载）
pnpm dev:turbo
```

🎉 访问 [http://localhost:3000](http://localhost:3000) 开始开发！

### 可用脚本


## 🐛 常见问题

### 数据库连接失败

**症状：** `PrismaClientInitializationError`

**解决方案：**
```bash
# 1. 检查网络连接
ping 100.113.231.101

# 2. 验证数据库配置
cat .env | grep DATABASE_URL

# 3. 重新生成Prisma客户端
pnpm db:generate
```

### Meilisearch连接失败

**症状：** 搜索功能无响应

**解决方案：**
```bash
# 1. 检查容器状态
docker ps | grep meilisearch

# 2. 重启容器
docker restart karakeep-meilisearch

# 3. 检查端口占用
lsof -i :7700
```

### 热重载不工作

**症状：** 修改代码后页面不自动刷新

**解决方案：**
```bash
# 1. 重启开发服务器
# Ctrl+C 停止，然后重新运行
pnpm dev

# 2. 尝试Turbopack
pnpm dev:turbo

# 3. 清理缓存
rm -rf .next && pnpm dev
```

### 端口冲突

**症状：** `EADDRINUSE: address already in use :::3000`

**解决方案：**
```bash
# 1. 找到占用端口的进程
lsof -i :3000

# 2. 终止进程
kill -9 <PID>

# 3. 或使用不同端口
PORT=3001 pnpm dev
```

## 🚀 性能优化建议

### 1. 开发环境优化

```bash
# 使用Turbopack提升构建速度
pnpm dev:turbo

# 禁用不必要的功能（在next.config.ts中）
experimental: {
  turbopack: true
}
```

### 2. 数据库查询优化

```typescript
// 使用Prisma的include和select优化查询
const content = await prisma.contents.findMany({
  select: {
    id: true,
    title: true,
    // 只选择需要的字段
  },
  include: {
    categories: true,
    // 只包含需要的关联数据
  }
});
```

### 3. 代码组织建议
