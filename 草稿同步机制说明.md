# 草稿同步机制说明

## 📋 核心概念

### 什么是"未入库"？

**未入库 = Karakeep 中存在，但本地数据库中还没有的书签**

判断标准：根据 `karakeep_id` 是否存在于本地 `drafts` 表中

### 什么是"已入库"？

**已入库 = 本地数据库中已有记录（无论 status 是什么）**

一旦有了 `status` 字段，说明这条书签已经同步过了

## 🔄 三种同步模式

### 1. 全局同步（发现新内容）

**作用**：同步 Karakeep 中的新书签

**逻辑**：
```
1. 从 Karakeep 获取所有书签（archived=false）
2. 查询本地数据库已有的 karakeep_id
3. 过滤：只保留本地不存在的书签
4. 创建新草稿（status=pending）
```

**API**：`POST /api/drafts/sync`

**使用场景**：
- 定期发现 Karakeep 中的新书签
- 首次使用系统时批量导入

**示例**：
```typescript
// 前端
const { mutateAsync } = useSyncDrafts();
await mutateAsync();

// 结果
同步完成（共 50 条）：
- 新增 10 条  ← 真正新增的书签
- 未变化 40 条 ← 本地已存在，跳过
```

### 2. 单独同步（刷新指定草稿）

**作用**：刷新某个已入库草稿的最新数据

**逻辑**：
```
1. 根据 draft.karakeep_id 从 Karakeep 获取最新数据
2. 更新本地草稿的内容
3. 保留本地字段（status, priority, category_suggestion）
4. 可选：将书签添加到 Karakeep 指定列表
```

**API**：`POST /api/drafts/:id/sync`

**使用场景**：
- 在 Karakeep 中编辑了某个书签
- 想立即获取该书签的最新内容
- AI 重新处理了书签，需要刷新总结/标签

**示例**：
```typescript
// 前端
const { mutateAsync } = useSyncSingleDraft();
await mutateAsync({
  id: draftId,
  addToList: 'weekly-list-id', // 可选
});

// 结果
{ success: true, message: "同步成功（已更新）" }
```

### 3. 批量同步（刷新选中草稿）

**作用**：批量刷新多个已入库草稿的数据

**逻辑**：
```
1. 遍历选中的草稿 ID 列表
2. 逐个调用单独同步
3. 统计成功/失败/更新/未变化的数量
```

**API**：`POST /api/drafts/sync-batch`

**使用场景**：
- 筛选出一批草稿，批量刷新
- 某些高优先级草稿需要定期更新

**示例**：
```typescript
// 前端
const { mutateAsync } = useSyncBatchDrafts();
await mutateAsync({
  draftIds: selectedIds,
  addToList: 'weekly-list-id', // 可选
});

// 结果
{
  total: 5,
  success: 5,
  failed: 0,
  updated: 3,    ← 内容有变化
  unchanged: 2   ← 内容无变化
}
```

## 🎯 完整工作流

```
Karakeep                          本地系统
   ├─ 收藏新书签 ──────────►  【全局同步】
   │   (100 条)                    ├─ 过滤已入库 (80 条)
   │                              └─ 新增 20 条草稿 ✅
   │
   ├─ 编辑书签内容 ────────►  【单独同步】
   │   (修改标题/笔记)             └─ 更新指定草稿 ✅
   │
   ├─ AI 重新处理 ─────────►  【批量同步】
   │   (优化总结/标签)             └─ 刷新选中草稿 ✅
   │
   └─ 采用/拒绝草稿 ◄────────  【双向同步】
       (自动归档)                  └─ archived=true
```

## 📊 字段说明

### processSingleBookmark 的行为

#### 创建模式（existing = null）

```typescript
// 全局同步时触发
if (!existing) {
  await prisma.drafts.create({
    data: {
      // Karakeep 数据
      karakeep_id,
      title,
      summary,
      tags_suggestion,
      // ...
      
      // 本地新增字段
      status: 'pending',        // 默认状态
      category_suggestion,      // 自动生成
      priority: 0,             // 默认优先级
    }
  });
}
```

#### 更新模式（existing != null）

```typescript
// 单独/批量同步时触发
if (existing) {
  if (hasChanges) {
    await prisma.drafts.update({
      data: {
        // 更新 Karakeep 相关字段
        title,
        summary,
        description,
        note,
        tags_suggestion,
        // ...
        
        // 保留本地管理字段（不更新）
        // status - 保持原值
        // priority - 保持原值
        // category_suggestion - 保持原值
        // content_id - 保持原值
      }
    });
  }
}
```

## 🔒 字段保护机制

### 永不更新的字段（本地管理）

- ✅ `status` - 草稿状态（pending/adopted/rejected）
- ✅ `priority` - 优先级（1-5星）
- ✅ `category_suggestion` - 分类建议（首次生成后不变）
- ✅ `content_id` - 关联的内容 ID
- ✅ `duplicate_of_draft_id` - 重复标记

### 同步时更新的字段（Karakeep 数据）

- 🔄 `title` - 标题
- 🔄 `url` - 链接
- 🔄 `description` - 描述
- 🔄 `summary` - AI 总结
- 🔄 `note` - 笔记
- 🔄 `tags_suggestion` - 标签建议（JSON）
- 🔄 `tagging_status` - 标注状态
- 🔄 `summarization_status` - 总结状态
- 🔄 `favicon_url` - 图标
- 🔄 `image_url` - 图片
- 🔄 `content` - 周刊格式内容
- 🔄 `slug` - URL 标识
- 🔄 `source` - 来源网站
- 🔄 `word_count` - 字数

## 💡 最佳实践

### 推荐工作流

```
每天早上：
1. 点击"全局同步"按钮
   ↓
   发现并导入新书签（10-20 条）
   ↓
2. 审核新草稿，设置优先级
   ↓
3. 勾选高优先级草稿
   ↓
4. 点击"批量同步"刷新最新数据
   ↓
5. 采用/拒绝草稿
   ↓
   自动归档到 Karakeep（archived=true）
```

### 避免重复同步

```
已采用/已拒绝的草稿
    ↓
自动归档 (archived=true)
    ↓
全局同步时会跳过 (archived=false 参数)
    ↓
不会重新导入 ✅
```

### 按需刷新

```
不需要全量刷新时：
- 单个草稿 → 点击"刷新"按钮
- 多个草稿 → 勾选后点击"批量同步"
- 节省时间，精准更新 ⚡
```

## 🚀 性能对比

| 场景 | 传统全局同步 | 优化后全局同步 | 提升 |
|------|------------|--------------|-----|
| 500 条书签（480 已入库） | 处理全部 500 条 | 只处理 20 条新书签 | 96% ⬆️ |
| 100 条书签（90 已入库） | 处理全部 100 条 | 只处理 10 条新书签 | 90% ⬆️ |
| 刷新 5 个草稿 | 全局同步 | 批量同步 5 个 | 95% ⬆️ |

## 🎓 总结

| 功能 | 作用 | 触发方式 | 数据源 |
|------|------|---------|-------|
| **全局同步** | 发现新书签 | 主动点击 | Karakeep 全部书签 → 过滤已入库 |
| **单独同步** | 刷新一个草稿 | 点击行内按钮 | Karakeep 单个书签 |
| **批量同步** | 刷新多个草稿 | 勾选后点击 | Karakeep 多个书签 |

**核心原则**：
- ✅ 全局同步只创建新的，不更新已有的
- ✅ 单独/批量同步只更新已有的，不创建新的
- ✅ 本地管理字段（status/priority）永远不会被覆盖
- ✅ 已处理的草稿自动归档，不会重复出现
