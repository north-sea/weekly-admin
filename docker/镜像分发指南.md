# 🐳 Docker镜像分发指南 - 生产环境

⚠️ **重要说明：本指南仅适用于生产环境部署！**

**本地开发不需要Docker，请使用 `pnpm dev` 直接运行！**

本指南详细说明了如何在生产环境中分发和部署Weekly内容管理系统的Docker镜像。

## 核心概念理解

### 1. Docker Registry机制

**什么是Docker Registry？**
Docker Registry是用于存储和分发Docker镜像的中央仓库，类似于代码的Git仓库。

**镜像命名规则：**
```
[registry-host[:port]/][username/]repository:tag
```

示例：
- `weekly-content-management:latest` (本地镜像)
- `registry.nas.local:5000/weekly-content-management:v1.2.3` (私有registry)
- `your-username/weekly-content-management:latest` (Docker Hub)

### 2. 环境变量解释

**在docker-compose.prod.yml中：**
```yaml
image: ${DOCKER_REGISTRY:-}weekly-content-management:${APP_VERSION:-latest}
```

**语法说明：**
- `${DOCKER_REGISTRY:-}`: 如果环境变量`DOCKER_REGISTRY`有值，则使用该值；否则为空字符串
- `${APP_VERSION:-latest}`: 如果环境变量`APP_VERSION`有值，则使用该值；否则默认为`latest`

**实际效果示例：**

| DOCKER_REGISTRY | APP_VERSION | 最终镜像名 |
|-----------------|-------------|------------|
| 未设置 | 未设置 | `weekly-content-management:latest` |
| `""` | `v1.2.3` | `weekly-content-management:v1.2.3` |
| `registry.nas.local:5000` | `latest` | `registry.nas.local:5000/weekly-content-management:latest` |
| `your-username` | `v2.0.0` | `your-username/weekly-content-management:v2.0.0` |

## 三种分发方案详解

### 方案一：私有Docker Registry（推荐）

**适用场景：** 团队开发、生产环境、需要版本管理

**优势：**
- ✅ 标准化流程
- ✅ 支持多版本管理  
- ✅ 自动化部署友好
- ✅ 安全可控

#### 1.1 快速搭建私有Registry

**在NAS服务器上运行：**
```bash
# 创建registry存储目录
mkdir -p /volume1/docker/registry

# 启动registry服务
docker run -d \
  --name weekly-registry \
  --restart=unless-stopped \
  -p 5000:5000 \
  -v /volume1/docker/registry:/var/lib/registry \
  registry:2
```

**配置访问（可选）：**
```bash
# 如果需要认证，创建密码文件
mkdir -p /volume1/docker/registry/auth
docker run --rm \
  registry:2 \
  htpasswd -bBn admin your-password > /volume1/docker/registry/auth/htpasswd

# 重启registry启用认证
docker stop weekly-registry && docker rm weekly-registry
docker run -d \
  --name weekly-registry \
  --restart=unless-stopped \
  -p 5000:5000 \
  -v /volume1/docker/registry:/var/lib/registry \
  -v /volume1/docker/registry/auth:/auth \
  -e "REGISTRY_AUTH=htpasswd" \
  -e "REGISTRY_AUTH_HTPASSWD_REALM=Registry Realm" \
  -e "REGISTRY_AUTH_HTPASSWD_PATH=/auth/htpasswd" \
  registry:2
```

#### 1.2 开发环境配置

**本地.env文件：**
```env
DOCKER_REGISTRY="100.113.231.101:5000"
APP_VERSION="v1.0.0"
```

**构建和推送：**
```bash
# 构建并推送到私有registry
DOCKER_REGISTRY="100.113.231.101:5000" ./scripts/build-and-push.sh v1.0.0
```

#### 1.3 生产环境部署

**NAS服务器.env文件：**
```env
DOCKER_REGISTRY="100.113.231.101:5000"
APP_VERSION="v1.0.0"
```

**部署命令：**
```bash
# 拉取并启动
docker compose -f docker/docker-compose.prod.yml pull
docker compose -f docker/docker-compose.prod.yml up -d
```

### 方案二：Docker save/load传输

**适用场景：** 单台服务器、临时部署、网络受限环境

**优势：**
- ✅ 无需额外服务
- ✅ 操作简单直接
- ⚠️ 手动操作较多

#### 2.1 本地构建和导出

```bash
# 构建镜像
docker build -f docker/Dockerfile -t weekly-content-management:v1.0.0 .

# 导出镜像
docker save weekly-content-management:v1.0.0 | gzip > weekly-admin-v1.0.0.tar.gz
```

#### 2.2 传输到NAS服务器

```bash
# 方式1: scp传输
scp weekly-admin-v1.0.0.tar.gz nas:/volume1/docker/images/

# 方式2: 通过NAS文件管理界面上传
# 将文件上传到 /volume1/docker/images/ 目录
```

#### 2.3 NAS服务器导入和运行

```bash
# SSH连接到NAS服务器
ssh nas

# 导入镜像
cd /volume1/docker/images/
gunzip -c weekly-admin-v1.0.0.tar.gz | docker load

# 更新.env文件
echo "APP_VERSION=v1.0.0" > .env
echo "DOCKER_REGISTRY=" >> .env

# 启动服务
docker compose -f docker/docker-compose.prod.yml up -d
```

### 方案三：NAS共享目录方式

**适用场景：** 利用现有NAS文件共享、自动化要求不高

#### 3.1 设置共享目录

**在NAS上创建共享目录：**
```bash
mkdir -p /volume1/shared/docker-images
```

#### 3.2 本地自动化脚本

创建`scripts/deploy-to-nas.sh`：
```bash
#!/bin/bash
set -e

VERSION=${1:-$(date +%Y%m%d-%H%M%S)}
NAS_MOUNT="/Volumes/shared"  # macOS挂载点

echo "🏗️ 构建镜像..."
docker build -f docker/Dockerfile -t weekly-content-management:${VERSION} .

echo "📦 导出镜像..."
docker save weekly-content-management:${VERSION} | gzip > ${NAS_MOUNT}/docker-images/weekly-admin-${VERSION}.tar.gz

echo "📝 创建部署脚本..."
cat > ${NAS_MOUNT}/docker-images/deploy-${VERSION}.sh << EOF
#!/bin/bash
cd /volume1/shared/docker-images
gunzip -c weekly-admin-${VERSION}.tar.gz | docker load
echo "APP_VERSION=${VERSION}" > /volume1/docker/weekly/.env
echo "DOCKER_REGISTRY=" >> /volume1/docker/weekly/.env
cd /volume1/docker/weekly
docker compose -f docker/docker-compose.prod.yml up -d
EOF

chmod +x ${NAS_MOUNT}/docker-images/deploy-${VERSION}.sh
echo "✅ 部署包已准备完成"
echo "在NAS上运行: bash /volume1/shared/docker-images/deploy-${VERSION}.sh"
```

## 故障排除

### 常见问题

1. **registry连接失败**
   ```bash
   # 检查registry服务状态
   curl http://100.113.231.101:5000/v2/
   
   # 检查防火墙和端口
   telnet 100.113.231.101 5000
   ```

2. **镜像拉取失败**
   ```bash
   # 登录到registry（如果需要认证）
   docker login 100.113.231.101:5000
   
   # 手动拉取测试
   docker pull 100.113.231.101:5000/weekly-content-management:latest
   ```

3. **镜像过大传输慢**
   ```bash
   # 使用multi-stage构建减小镜像体积
   # 或者使用镜像压缩
   docker save image:tag | gzip > image.tar.gz
   ```
