# 🚀 Weekly内容管理系统 - 生产环境部署指南

⚠️ **重要说明：本指南仅适用于生产环境部署！**

**本地开发请使用：`pnpm dev`，无需Docker！**

## 环境变量快速参考

### 🎯 核心概念

**docker-compose.prod.yml中的镜像名称：**
```yaml
image: ${DOCKER_REGISTRY:-}weekly-content-management:${APP_VERSION:-latest}
```

**实际效果：**

| DOCKER_REGISTRY | APP_VERSION | 最终镜像名 |
|-----------------|-------------|------------|
| `""` (空) | `latest` | `weekly-content-management:latest` |
| `""` (空) | `v1.2.3` | `weekly-content-management:v1.2.3` |
| `registry.nas.local:5000` | `latest` | `registry.nas.local:5000/weekly-content-management:latest` |
| `your-username` | `v2.0.0` | `your-username/weekly-content-management:v2.0.0` |

## 🔧 必需的.env文件内容

**最小配置：**
```env
# 数据库连接 (必需)
DATABASE_URL="mysql://weekly_user:weekly_20250629_@NAS@100.113.231.101:3306/weekly_blog"

# 搜索引擎 (必需) 
MEILISEARCH_HOST="http://localhost:7700"
MEILISEARCH_MASTER_KEY="your_master_key"

# JWT认证 (必需)
JWT_SECRET="your-super-secret-jwt-key-please-change-this"
JWT_EXPIRES_IN="8h"

# Docker配置 (部署必需)
DOCKER_REGISTRY=""  # 留空使用本地镜像
APP_VERSION="latest"
APP_PORT="3000"
NODE_ENV="production"
```

## 📋 三种部署方案

### 方案1: 本地构建 + 手动传输 (推荐入门)

**适用场景：** 单台NAS服务器，生产环境简单部署

**开发机操作：**
```bash
# 1. 构建镜像
./scripts/build-and-push.sh v1.0.0

# 2. 导出镜像
docker save weekly-content-management:v1.0.0 | gzip > weekly-admin-v1.0.0.tar.gz

# 3. 传输到NAS
scp weekly-admin-v1.0.0.tar.gz nas:/tmp/
```

**NAS服务器操作：**
```bash
# 4. 导入镜像
gunzip -c /tmp/weekly-admin-v1.0.0.tar.gz | docker load

# 5. 配置环境变量
# 从Docker配置模板复制
cp docker/env.example .env

# 编辑关键配置
cat > .env << EOF
DATABASE_URL="mysql://weekly_user:weekly_20250629_@NAS@100.113.231.101:3306/weekly_blog"
MEILISEARCH_HOST="http://karakeep-meilisearch:7700"
MEILISEARCH_MASTER_KEY="your_master_key"
JWT_SECRET="production-super-secret-jwt-key"
DOCKER_REGISTRY=""
APP_VERSION="v1.0.0"
APP_PORT="3000"
NODE_ENV="production"
EOF

# 6. 启动服务
docker compose -f docker/docker-compose.prod.yml up -d
```

### 方案2: 私有Registry (推荐生产)

**适用场景：** 团队开发，自动化部署

**1. 在NAS上搭建Registry：**
```bash
# 创建registry容器
docker run -d --name weekly-registry --restart=unless-stopped \
  -p 5000:5000 \
  -v /volume1/docker/registry:/var/lib/registry \
  registry:2
```

**2. 开发机配置：**
```bash
# 构建并推送
DOCKER_REGISTRY="100.113.231.101:5000" ./scripts/build-and-push.sh v1.0.0
```

**3. NAS服务器配置：**
```bash
# 复制Docker部署配置模板
cp docker/env.example .env

# 编辑配置文件
cat > .env << EOF
DATABASE_URL="mysql://weekly_user:weekly_20250629_@NAS@100.113.231.101:3306/weekly_blog"
MEILISEARCH_HOST="http://karakeep-meilisearch:7700" 
MEILISEARCH_MASTER_KEY="your_master_key"
JWT_SECRET="production-super-secret-jwt-key"
DOCKER_REGISTRY="100.113.231.101:5000"
APP_VERSION="v1.0.0"
APP_PORT="3000"
NODE_ENV="production"
EOF

# 拉取并启动
docker compose -f docker/docker-compose.prod.yml pull
docker compose -f docker/docker-compose.prod.yml up -d
```

### 方案3: Docker Hub公共仓库

**适用场景：** 开源项目，公共访问

**开发机操作：**
```bash
# 登录Docker Hub
docker login

# 构建并推送
DOCKER_REGISTRY="your-username" ./scripts/build-and-push.sh v1.0.0
```

**NAS服务器.env：**
```bash
# 复制并编辑Docker配置模板
cp docker/env.example .env
vim .env

# 配置示例：
# DOCKER_REGISTRY="your-username"
# APP_VERSION="v1.0.0"
# MEILISEARCH_HOST="http://karakeep-meilisearch:7700"
```

## 🔄 日常更新流程

### 发布新版本
```bash
# 开发机
./scripts/build-and-push.sh v1.2.3
```

### 更新NAS服务器
```bash
# NAS服务器
echo "APP_VERSION=v1.2.3" >> .env
docker compose -f docker/docker-compose.prod.yml pull
docker compose -f docker/docker-compose.prod.yml up -d
```

## 🛠️ 故障排除

### 常见问题

**1. 镜像拉取失败**
```bash
# 检查镜像是否存在
docker images | grep weekly-content-management

# 手动拉取测试
docker pull registry.nas.local:5000/weekly-content-management:latest
```

**2. 环境变量不生效**
```bash
# 检查.env文件
cat .env

# 验证compose配置
docker compose -f docker/docker-compose.prod.yml config
```

**3. 服务启动失败**
```bash
# 查看日志
docker compose -f docker/docker-compose.prod.yml logs

# 检查容器状态
docker compose -f docker/docker-compose.prod.yml ps
```

## 📞 快速命令参考

### 构建和部署
```bash
# 本地构建
DOCKER_REGISTRY="" ./scripts/build-and-push.sh v1.0.0

# Registry推送
DOCKER_REGISTRY="registry.nas.local:5000" ./scripts/build-and-push.sh v1.0.0

# 启动服务
docker compose -f docker/docker-compose.prod.yml up -d

# 更新服务
docker compose -f docker/docker-compose.prod.yml pull && \
docker compose -f docker/docker-compose.prod.yml up -d
```

### 监控和维护
```bash
# 查看状态
docker compose -f docker/docker-compose.prod.yml ps

# 查看日志
docker compose -f docker/docker-compose.prod.yml logs -f

# 重启服务
docker compose -f docker/docker-compose.prod.yml restart

# 停止服务
docker compose -f docker/docker-compose.prod.yml down
```
