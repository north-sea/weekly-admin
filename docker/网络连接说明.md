# Docker容器网络连接说明

## 🌐 当前网络架构

### 现有服务
1. **karakeep-meilisearch容器**
   - 网络模式：bridge (默认)
   - 端口映射：7700:7700
   - 容器名：karakeep-meilisearch

2. **weekly-content-app容器**（部署时创建）
   - 网络模式：bridge (通过docker-compose.prod.yml)
   - 端口映射：3000:3000

3. **MySQL数据库**
   - 位置：NAS服务器 (100.113.231.101:3306)
   - 连接方式：外部网络连接

## 🔗 网络连接方式

### 方式1: 容器名连接（推荐）
```env
MEILISEARCH_HOST="http://karakeep-meilisearch:7700"
```

**优势：**
- Docker自动DNS解析
- 不依赖固定IP
- 性能更好（容器间直接通信）

**要求：**
- 两个容器在同一网络或可通信的网络中
- 使用默认bridge网络时通常可以工作

### 方式2: 主机端口连接
```env
MEILISEARCH_HOST="http://host.docker.internal:7700"
```

**适用场景：**
- 容器名连接失败时的备选方案
- 跨网络连接

### 方式3: 本地端口连接
```env
MEILISEARCH_HOST="http://localhost:7700"
```

**适用场景：**
- 使用host网络模式时
- 特定网络配置下

## 🧪 网络连接测试

### 1. 验证容器运行状态
```bash
# 检查meilisearch容器状态
docker ps | grep karakeep-meilisearch

# 检查应用容器状态（部署后）
docker ps | grep weekly-content-app
```

### 2. 测试网络连通性
```bash
# 从应用容器ping meilisearch容器
docker exec weekly-content-app ping karakeep-meilisearch

# 测试HTTP连接
docker exec weekly-content-app curl http://karakeep-meilisearch:7700/health
```

### 3. 测试端口访问
```bash
# 从主机测试meilisearch端口
curl http://localhost:7700/health

# 从容器内测试
docker exec weekly-content-app curl http://localhost:7700/health
```

## 🔧 故障排除

### 问题1: 容器名解析失败
**错误信息：** `getaddrinfo ENOTFOUND karakeep-meilisearch`

**解决方案：**
```bash
# 选项1: 确保容器在同一网络
docker network ls
docker inspect karakeep-meilisearch | grep NetworkMode
docker inspect weekly-content-app | grep NetworkMode

# 选项2: 改用备选连接方式
MEILISEARCH_HOST="http://host.docker.internal:7700"

# 选项3: 创建自定义网络
docker network create weekly-network
docker network connect weekly-network karakeep-meilisearch
# 在docker-compose.prod.yml中使用weekly-network
```

### 问题2: 端口连接被拒绝
**错误信息：** `connect ECONNREFUSED`

**解决方案：**
```bash
# 检查meilisearch容器是否运行
docker ps | grep karakeep-meilisearch

# 检查端口映射
docker port karakeep-meilisearch

# 重启meilisearch容器
docker restart karakeep-meilisearch
```

### 问题3: 网络策略限制
**解决方案：**
```bash
# 使用host网络模式（如果安全策略允许）
# 在docker-compose.prod.yml中添加：
# network_mode: "host"

# 或创建专用网络并连接所有相关容器
docker network create --driver bridge weekly-bridge
docker network connect weekly-bridge karakeep-meilisearch
```

## 📋 推荐网络配置

### 生产环境推荐配置
```yaml
# docker-compose.prod.yml
version: '3.8'
services:
  app:
    # ... 其他配置
    networks:
      - weekly-network
    depends_on:
      - meilisearch  # 如果meilisearch也在compose中

networks:
  weekly-network:
    external: true
    name: ${EXISTING_NETWORK_NAME:-bridge}
```

### 环境变量推荐配置
```env
# 推荐：容器名连接
MEILISEARCH_HOST="http://karakeep-meilisearch:7700"

# 备选：主机端口连接
# MEILISEARCH_HOST="http://host.docker.internal:7700"

# 备选：本地端口连接
# MEILISEARCH_HOST="http://localhost:7700"
```

## 🔍 调试命令

### 网络诊断
```bash
# 查看所有网络
docker network ls

# 查看特定网络详情
docker network inspect bridge

# 查看容器网络配置
docker inspect <container_name> | grep -A 20 "NetworkSettings"

# 测试容器间连通性
docker exec <container1> ping <container2>
docker exec <container1> telnet <container2> <port>
```

### 日志检查
```bash
# 查看应用容器日志
docker logs weekly-content-app

# 查看meilisearch容器日志
docker logs karakeep-meilisearch

# 实时日志
docker logs -f weekly-content-app
```
