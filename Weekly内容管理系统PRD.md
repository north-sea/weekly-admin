## 1. 产品概述

### 1.1 产品定位和愿景

**产品定位**：专业的周刊内容管理系统，为内容创作者提供高效的内容聚合、编辑和发布工具。

**产品愿景**：打造一个简洁、高效、专业的内容管理平台，让周刊内容的管理变得轻松而有序。

**核心价值**：
- **效率优先**：简化内容管理流程，提升工作效率
- **专业工具**：提供专业的Markdown编辑和预览功能
- **灵活管理**：支持多状态内容管理和灵活的分类系统
- **优雅体验**：现代化的界面设计和流畅的交互体验

### 1.2 目标用户画像

**主要用户**：内容创作者和编辑人员
- **使用场景**：定期发布周刊内容，管理大量文章和资源链接
- **技术背景**：熟悉Markdown语法，具备基础的内容管理经验
- **核心需求**：高效的内容组织、便捷的编辑工具、直观的预览功能

**次要用户**：系统管理员
- **使用场景**：用户管理、系统维护、数据分析
- **技术背景**：具备一定的技术背景和系统管理经验
- **核心需求**：系统稳定性、数据安全性、管理便捷性

### 1.3 产品架构概览

```
┌─────────────────────┐
│    用户界面层       │
├─────────────────────┤
│    业务逻辑层       │
├─────────────────────┤
│    数据访问层       │
├─────────────────────┤
│    数据存储层       │
└─────────────────────┘
```

**技术栈**：
- 前端：Next.js 15 + TypeScript + Ant Design
- 后端：Next.js API Routes
- 数据库：MySQL
- 部署：NAS服务器

## 2. 功能需求规格

### 2.1 用户认证系统

#### 2.1.1 登录功能
**功能描述**：提供安全的用户登录机制

**具体需求**：
- 用户名/密码登录方式
- "记住我"功能，支持30天免登录
- 登录失败次数限制（5次后锁定10分钟）
- JWT token管理，会话超时控制（8小时）

**用户流程**：
1. 用户输入用户名和密码
2. 系统验证凭据
3. 登录成功后跳转到仪表板
4. 登录失败显示错误提示

#### 2.1.2 用户管理
**功能描述**：管理系统用户账户

**数据结构需求**：
```sql
CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    display_name VARCHAR(100),
    role ENUM('admin', 'editor') DEFAULT 'editor',
    last_login_at TIMESTAMP NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

### 2.2 内容管理模块

#### 2.2.1 内容列表页面
**功能描述**：统一展示和管理Blog和Weekly两种类型的内容条目

**核心功能**：
- **多视图模式**：卡片视图、表格视图
- **内容类型筛选**：全部、Blog、Weekly
- **状态筛选**：草稿、已发布、已归档、隐藏
- **搜索功能**：标题、内容、标签全文搜索
- **批量操作**：批量删除、批量修改状态、批量分类
- **排序功能**：按创建时间、更新时间、浏览量排序

**界面要求**：
- 内容类型使用不同的图标标识（📝 Blog, 📰 Weekly）
- 状态使用不同颜色的标签显示
- Weekly内容显示来源信息，Blog内容显示描述摘要
- 支持无限滚动或分页（每页20条）
- 响应式设计，适配不同屏幕尺寸

#### 2.2.2 内容编辑功能
**功能描述**：提供差异化的内容编辑能力，根据内容类型显示不同的字段

**编辑器规格**：
- **Markdown编辑器**：语法高亮、实时预览
- **分屏模式**：左侧编辑，右侧预览
- **快捷功能**：插入链接、图片、代码块
- **自动保存**：每30秒自动保存草稿

**通用编辑字段**：
- 内容类型（Blog/Weekly，创建时选择，不可修改）
- 标题（必填，最多500字符）
- 内容正文（支持Markdown/MDX格式）
- 分类选择（单选）
- 标签管理（多选，支持新建标签）
- SEO设置（meta标题、描述、关键词）
- 发布设置（状态、发布时间）

**Blog类型专用字段**：
- 描述/摘要（必填，最多1000字符）
- 阅读时间（自动计算）
- 封面图片（可选）

**Weekly类型专用字段**：
- 来源名称（必填，最多200字符）
- 来源链接（必填，URL格式验证）
- 截图API类型（ScreenshotLayer/HCTI/manual）
- 推荐理由（可选，最多500字符）

#### 2.2.3 内容预览功能
**功能描述**：提供内容的实时预览

**预览模式**：
- **编辑器预览**：编辑时的实时预览
- **独立页面预览**：完整的内容展示效果
- **移动端预览**：模拟移动设备显示效果

### 2.3 周刊管理模块

#### 2.3.1 周刊期号管理
**功能描述**：管理周刊的期号和发布，专门聚合Weekly类型的内容

**核心功能**：
- **期号创建**：自动递增期号、设置时间范围
- **Weekly内容选择**：仅从已发布的Weekly类型内容中选择
- **内容排序**：拖拽排序，支持分类分组
- **批量导入**：按时间范围批量导入Weekly内容
- **来源统计**：显示各来源网站的内容数量

**工作流程**：
1. 创建新期号（设置期号、标题、时间范围）
2. 筛选Weekly内容（按状态=published + content_type_id=4）
3. 内容排序（按分类或自定义排序）
4. 预览确认（查看整期效果，包含来源信息）
5. 发布周刊（设置发布时间）

#### 2.3.2 周刊编辑界面
**功能描述**：提供周刊内容的组织和编辑，专注于Weekly内容的聚合

**界面设计**：
- **左侧面板**：可选Weekly内容列表（按分类分组，显示来源信息）
- **中间区域**：已选内容列表（支持拖拽排序，显示来源和截图）
- **右侧面板**：实时预览效果

**操作功能**：
- 拖拽添加Weekly内容到周刊
- 拖拽调整内容顺序
- 快速删除不需要的内容
- 分类折叠/展开显示
- 按来源筛选内容
- 显示内容截图（如果有）

**是否需要独立Blog管理？**

**支持独立管理的理由**：
- Blog和Weekly有不同的使用场景和工作流
- Blog更注重长文写作和SEO优化

- 现有的内容管理模块已经支持类型差异化
- 避免功能重复，减少维护成本
- 用户可以通过内容类型筛选来管理Blog

### 2.6 高级搜索和筛选系统

#### 2.6.1 全文搜索功能
**功能描述**：基于Meilisearch提供强大的内容检索能力

**核心功能**：
- **全文检索**：支持标题、描述、正文内容的关键词搜索
- **即时搜索**：search-as-you-type，输入即搜索
- **搜索高亮**：搜索结果中高亮显示匹配的关键词
- **搜索建议**：智能搜索建议和自动补全
- **中文分词**：支持中文内容的精确分词搜索
- **搜索历史**：保存用户的搜索历史记录

**搜索范围**：
- 按内容类型搜索（Blog/Weekly）
- 按分类搜索
- 按标签搜索
- 按状态搜索
- 按时间范围搜索
- 按来源搜索（Weekly专用）

**搜索特性**：
- **相关性排序**：基于内容相关性智能排序
- **模糊搜索**：容错搜索，支持拼写错误
- **同义词支持**：配置同义词提升搜索覆盖率
- **搜索统计**：搜索热词统计和分析

#### 2.6.2 高级筛选器
**功能描述**：多维度组合筛选功能

**筛选维度**：
- **内容类型**：Blog/Weekly
- **发布状态**：draft/published/archived/hidden
- **分类筛选**：支持多选分类
- **标签筛选**：支持多选标签，AND/OR逻辑
- **时间筛选**：创建时间、更新时间、发布时间范围
- **来源筛选**：按来源网站筛选（Weekly专用）
- **字数筛选**：按内容字数范围筛选

**筛选交互**：
- 筛选条件保存和快速应用
- 筛选结果数量实时显示
- 一键清除所有筛选条件
- 筛选条件的URL持久化（可分享）

### 2.7 操作日志系统

#### 2.7.1 操作记录功能
**功能描述**：记录系统内所有重要操作，确保可追溯性

**记录内容**：
- **用户操作**：登录、退出、密码修改
- **内容操作**：创建、编辑、删除、状态变更
- **周刊操作**：创建期号、添加内容、发布
- **管理操作**：标签合并、分类管理、用户管理

**日志字段**：
- 操作时间（精确到秒）
- 操作用户（用户ID和用户名）
- 操作类型（CREATE/UPDATE/DELETE等）
- 操作对象（内容ID、周刊ID等）
- 操作详情（具体变更内容）
- IP地址和用户代理

#### 2.7.2 版本历史功能
**功能描述**：保存内容的修改历史，支持版本对比和回滚

**版本管理**：
- **自动保存**：每次内容修改自动创建版本
- **版本对比**：可视化显示两个版本之间的差异
- **版本回滚**：一键恢复到历史版本
- **版本清理**：定期清理过期的历史版本

**版本信息**：
- 版本号（自动递增）
- 修改时间和修改人
- 变更摘要（字段级别的变更说明）
- 版本标签（可手动标记重要版本）

#### 2.7.3 日志查询和分析
**功能描述**：提供操作日志的查询和分析功能

**查询功能**：
- 按用户查询操作记录
- 按时间范围查询
- 按操作类型筛选
- 按操作对象搜索

**分析功能**：
- 用户活跃度统计
- 操作频率分析
- 异常操作检测
- 系统使用趋势

### 2.8 系统管理功能

#### 2.8.1 标签管理模块
**功能描述**：完整的标签生命周期管理

**核心功能**：
- **标签CRUD操作**：
  - 创建标签：名称+路径标识验证
  - 编辑标签：修改名称和路径标识
  - 删除标签：检查使用情况，安全删除
  - 查看标签：显示使用次数和关联内容

- **标签合并功能**：
  - 选择多个标签进行合并
  - 指定保留的主标签
  - 自动更新content_tags表中的关联关系
  - 重新计算标签使用次数

- **批量操作**：
  - 批量删除未使用的标签
  - 批量重命名相似标签
  - 标签使用情况统计和清理建议

**数据一致性保证**：
- 删除标签时自动清理content_tags关联记录
- 合并标签时保证所有内容关联不丢失
- 实时更新标签使用计数器

#### 2.6.2 分类管理模块
**功能描述**：支持层级结构的分类管理系统

**核心功能**：
- **分类CRUD操作**：
  - 创建分类：支持父子关系设置
  - 编辑分类：修改名称、描述、排序权重
  - 删除分类：处理子分类和内容重新分配
  - 移动分类：调整分类层级结构

- **层级管理**：
  - 拖拽调整分类层级关系
  - 批量移动子分类
  - 分类树状结构显示
  - 层级深度限制（建议最多3级）

- **分类重组**：
  - 合并相似分类
  - 分类使用情况分析
  - 内容重新分类建议

**数据一致性保证**：
- 删除分类时处理子分类（设为NULL或转移到父分类）
- 删除分类时处理关联内容（转移到其他分类或设为未分类）
- 维护分类排序权重的一致性

## 3. 技术规格

### 3.1 技术架构设计

#### 3.1.1 整体架构
```
┌─────────────────────────────────────┐
│             前端层                  │
│  Next.js + TypeScript + Ant Design │
├─────────────────────────────────────┤
│             API层                   │
│         Next.js API Routes          │
├─────────────────────────────────────┤
│           数据访问层                │
│         Prisma ORM / Drizzle        │
├─────────────────────────────────────┤
│           数据存储层                │
│             MySQL 8.0               │
└─────────────────────────────────────┘
```

#### 3.1.2 技术选型说明

**前端技术栈**：
- **Next.js 15**：React全栈框架，支持SSR/SSG
- **TypeScript**：类型安全，提升开发效率
- **Ant Design + Pro-component**：企业级UI组件库
- **React Hook Form + Zod**：表单处理和验证
- **React Query**：数据获取和缓存管理

**后端技术栈**：
- **Next.js API Routes**：基于文件系统的API路由
- **Prisma ORM**：类型安全的数据库ORM
- **Meilisearch**：高性能全文搜索引擎
- **bcryptjs**：密码加密
- **jsonwebtoken**：JWT token管理
